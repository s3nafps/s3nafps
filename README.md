## Working Style

I prefer evidence over vague claims:

- before/after screenshots,
- exact commands checked,
- root-cause notes,
- restore-test proof,
- monitoring status,
- clear next actions.

No passwords, private keys, API tokens, or production `.env` files should be shared in chat, tickets, screenshots, or reports.
